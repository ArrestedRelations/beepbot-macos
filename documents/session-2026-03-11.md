### Changes to sidecar (`sidecar/src/index.ts`)

Added CORS headers via a Fastify `onRequest` hook so the browser at `localhost:3003` can make REST calls to the sidecar at `127.0.0.1:3004` without being blocked by the browser's same-origin policy. WebSocket connections are unaffected (WS is not subject to CORS).

```ts
app.addHook('onRequest', async (_req, reply) => {
  reply.header('Access-Control-Allow-Origin', '*');
  reply.header('Access-Control-Allow-Methods', 'GET, POST, PUT, PATCH, DELETE, OPTIONS');
  reply.header('Access-Control-Allow-Headers', 'Content-Type');
});
app.options('*', async (_req, reply) => { reply.send(); });
```

### How to run

```bash
# Tauri chatbot (as before)
pnpm tauri dev          # from beepbot/

# Dashboard (separate terminal)
cd dashboard && pnpm dev   # serves http://localhost:3003
```

Click the expand icon (Maximize2) in the Tauri bot to open the dashboard in the default browser.

## 5. Vite `allowedHosts` fix

Vite 7 blocks requests from unrecognized hostnames by default. Accessing `http://beepbotai:7432` returned "Blocked request. This host is not allowed."

**Fix:** Added `allowedHosts: ["beepbotai"]` to `dashboard/vite.config.ts` server config.
